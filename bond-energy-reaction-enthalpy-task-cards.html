键能与反应焓变

用化学键的视角看 ΔH · 算反应焓变的第三种方法
模块名称键能与反应焓变（基础模块）
所属大主题热力学 · 物理化学融合主题
预计学习时间2.5-3 小时（按自己的节奏）

盖斯定律（必读 —— 键能算反应 ΔH 本质是盖斯定律的另一种入口: 反应路径 = 全部断键到原子态 + 全部成键到产物）
所需工具AI 工具、纸笔、计算器；化学手册或 NIST WebBook 上的平均键能数据表；常见小分子（H₂ / O₂ / N₂ / CH₄ / H₂O / CO₂ 等）的化学键类型分析能力

学习目标

  1. 能陈述键能的严格定义（25°C、1 atm 下气态分子中 1 mol 化学键完全断裂吸收的能量，\(\Delta H_键 > 0\)），并解释 4 个关键限定词（气态 / 1 mol 键 / 完全断裂 / 吸热）
  2. 能区分离解能（精确单步键能）与平均键能（手册数据），以 H₂O 中的 O-H 键为例说明两者数值差异
  3. 能默写 5-8 种典型化学键的键能数据（C-H / O-H / C-C / C=C / C=O / O=O / H-H / N≡N 等）
  4. 能用键能算法 ΔH ≈ Σ(反应物键能) − Σ(产物键能) 算简单反应的 ΔH（如 H₂+Cl₂→2HCl / CH₄+2O₂→CO₂+2H₂O(g)）
  5. 能解释键能算法的 4 个使用限制（全气态 / 平均键能 ±5-10 kJ/mol 误差 / 不适用离子反应 / 必须完整断键+完整成键）
  6. 能在 ΔH 算法的三种方法（实验测量 / 盖斯定律+ΔHf° / 键能算法）之间根据反应特点选择最合适的方法
  7. 知道本板块不深入：键能的温度依赖 / 量子化学计算 / 离子反应的键焓 —— 这些留给进阶板块或大学物理化学
驱动问题

三个关于反应能量"微观本质"的核心问题 ——

  • 反应放热的微观原因：一个反应放热是因为产物的总能量比反应物低，但能量去了哪儿？反应过程中"被释放的能量"实际上是什么物理量的变化？
  • 不查实验数据能否估算 ΔH：盖斯定律 + ΔHf° 表给了我们算 ΔH 的方法 —— 但如果反应物 / 产物的 ΔHf° 在手册上查不到怎么办？有没有用更基本的"化学键"就能估算 ΔH 的方法？
  • 三种 ΔH 算法的关系：到目前为止我们学过实验测（燃烧热 / 中和热）和盖斯定律算（用 ΔHf°）两种方法。它们是否有第三种方法？三种方法谁更精确？什么时候选谁？

本板块的任务，是引入第三种 ΔH 算法 —— 用化学键键能，并将三种方法整合成一个完整的"反应焓变 ΔH 算法体系"。

（提示：从"化学键"的视角看反应，反应 = 断键（吸热）+ 成键（放热）。这是化学反应能量的微观本质 —— 比盖斯定律 + ΔHf° 更"基本"。）

键能 — 定义 + 微观物理意义

▼
学习目标: 严格陈述键能定义（含 4 个关键限定词）→ 理解键能 = 断键吸热的微观意义 → 区分离解能与平均键能 → 掌握典型化学键键能数据
建议时长: 40-50 分钟
STEP 1先自己想
5-8 分钟

不查笔记，先答 ——

  • 一个化学反应放热的"微观原因"是什么？反应物的什么变成了产物的什么，差额变成了热量？
  • "断开化学键"和"形成化学键"，哪个吸热？哪个放热？为什么？
  • 如果给你 H-H 键、Cl-Cl 键、H-Cl 键的键能数值，你能算出 H₂ + Cl₂ → 2HCl 这个反应是吸热还是放热吗？
先回顾（核心理论根基）：
前置 热力学第一定律 板块讲过 —— 在恒压条件下系统吸收的热量 Qp 恰好等于焓变 ΔH，即 Qp = ΔH。化学反应的能量收支最终体现为 ΔH 的符号（放热 ΔH < 0，吸热 ΔH > 0）。
前置 盖斯定律 板块讲过 —— 反应焓变 ΔH 是状态量，只依赖始末态，与反应路径无关。这意味着我们可以选一条方便算的虚拟路径来算 ΔH，最终结果与真实路径一致。
本板块的键能算法，就是用一条特殊的虚拟路径 —— "反应物完全断键到原子态 + 原子态完全成键到产物" —— 来算 ΔH。
STEP 2键能的严格定义
键能定义 ⭐⭐

在标准条件（25°C、1 atm）下，气态分子中 1 mol 化学键完全断裂所吸收的能量，称为该化学键的键能，记作 \(\Delta H_键\)（单位：kJ/mol），数值始终为正（因为断键需要吸热）。

定义里的 4 个关键限定词:

限定词含义不这么规约会怎样？
气态分子键能针对"气相"中的化学键 — 排除分子间相互作用（液固相内的氢键 / 范德华力等）液固相内分子间有额外结合能，断键数值会偏高，且与分子间相互作用混在一起无法分离
1 mol 化学键归一化到"每摩尔键"不规定 mol 数，数值随键数变化，无法与单键能直接比较
完全断裂断键彻底分到单个原子（气态原子态），不是断到自由基或离子"半断裂"或"成离子"产物种类不同，数值各异，没法标准化
吸热（ΔH > 0）断键始终吸热（化学键的本质是稳定结合状态，破坏它必须输入能量）不强调"吸热"，容易把"键能"和"反应焓变"符号搞混（反应 ΔH 可正可负）
键能的微观物理意义: 化学键是原子间通过电子云重叠形成的稳定结合状态。"断键"= 把两个键合原子拉到无限远（不再相互作用）所需的能量。"成键"= 反向过程，放出同样多的能量。
· 单键能小（如 C-C 348）→ 键较弱，容易断
· 双键 / 三键能大（如 C=C 614 / N≡N 945）→ 键较强，难断
· 极性键能大于非极性键（如 H-F 565 > H-H 436）→ 电负性差使电子云分布更稳定

典型化学键键能数据（标准平均键能, kJ/mol, 25°C/1 atm 气相）:

化学键键能 (kJ/mol)说明
H-H436氢气分子单键 — 单原子单键，离解能
Cl-Cl243氯气分子单键 — 较弱的单键
H-Cl431极性单键，介于 H-H 和 Cl-Cl 之间
O=O498氧气分子双键 — 注意双键能不一定是单键的 2 倍
N≡N945氮气分子三键 — 极强，氮气难活化的原因
C-C348有机化合物的主链单键
C=C614烯烃双键
C-H413有机化合物 C-H 平均键能
O-H463水分子 / 醇 O-H 平均键能
C=O745CO₂ 中的 C=O / 羰基 C=O 平均

注: 上表是平均键能(来自多种含该键化合物的统计平均)，是估算反应 ΔH 用的"通用数据"。具体某分子中的某根键的实际离解能可能与平均键能差几十 kJ/mol。

STEP 3离解能 vs 平均键能 — 用 H₂O 看差异

"H₂O 的 O-H 键能"到底是多少？化学手册上常写 463 kJ/mol，但严格测量发现 —— H₂O 里两根 O-H 键的断裂能不一样。

H₂O 两根 O-H 键的逐步断裂
第 1 根 O-H 断裂: H₂O(g) → H(g) + OH(g)，ΔH₁ ≈ +498 kJ/mol(称为第一离解能)
第 2 根 O-H 断裂: OH(g) → O(g) + H(g)，ΔH₂ ≈ +428 kJ/mol(称为第二离解能)
两步总和 ΔH: H₂O(g) → 2H(g) + O(g)，ΔH = 498 + 428 = +926 kJ/mol
平均 O-H 键能: 926 / 2 = +463 kJ/mol — 这就是化学手册中常用的"O-H 平均键能"
结论 ⭐: 平均键能 ≠ 离解能。平均键能是"分子里所有同类键能的算术平均", 是工程估算用的"近似值"; 真实分子中每根键的离解能可能差几十 kJ/mol。
思考点 1 · 为什么键能定义要求"气态" + "完全断裂"？
键能的严格定义里规定"气态分子" + "完全断裂到原子态"。如果改成"液态分子"或"断到自由基"行不行？为什么？
两个限定词的物理意义不同 ——
气态: 液固相内分子间有额外结合能(氢键 / 范德华力 / 离子键)，断键时这些力也要克服 → 测得的"键能"数值偏高，且无法区分是化学键能还是分子间作用力。气态分子之间相距很远，分子间力 ≈ 0，断键只对应化学键本身的能量。
完全断裂到原子态: 如果只断到自由基(如 ·CH₃)，自由基本身还有未配对电子 / 自由价，能量与单原子(C/H)不同 → 数值不可比。原子态是"绝对的最高能量参考点"，所有键能在这个参考下统一。
这两个限定词一起把"键能"标准化到一个普适、可比、可加和的物理量，否则各种条件下测的数值无法整合到一个手册里。
在纸上: ① 写"气态"的物理理由(分子间作用力 = 0); ② 写"完全断裂到原子态"的物理理由(参考点统一); ③ 用 H₂O 液态 vs 气态举例说明"液态 O-H 键能"测出会偏高。
思考点 2 · 离解能 vs 平均键能 — 差几十 kJ/mol 算大还是小？
H₂O 的第一 O-H 离解能 498，第二 O-H 离解能 428，差 70 kJ/mol。但通常用平均键能 463 kJ/mol 算反应 ΔH。这个"折中"会带来多大误差？为什么实际工程上仍然可用？
70 kJ/mol 是单根键的差异，看似很大。但实际算反应 ΔH 时 ——
· 一个反应通常涉及多根同类键的断成(如 CH₄ 燃烧涉及 4 根 C-H + 2 根 O=O 断键 + 2 根 C=O + 4 根 O-H 成键)
· 用平均键能算时，"高估"和"低估"的误差会部分抵消(统计意义上的平均)
· 最终 ΔH 的误差通常控制在 ±5-10 kJ/mol(相对反应总能量几百到上千 kJ/mol，误差率 1-3%)
这是为什么平均键能虽然不严格，但仍是"工程估算"和"未知反应初步预判"的好工具。当需要更高精度时再用 ΔHf° 或实验测量。
在纸上: ① 写"为什么单键误差大但反应误差可控"(误差抵消); ② 一句话区分"平均键能"和"离解能"的使用场景(工程估算 vs 精确推导)。
离解能的精确推导(用量子化学计算 / 光谱实验)留给大学物理化学。本板块用平均键能即可。

▼
学习目标: 推导键能算法公式 ΔH ≈ Σ(反应物键能) − Σ(产物键能) → 用 3 个典型例题练习 → 理解键能算法的 4 个使用限制
建议时长: 40-50 分钟
STEP 1先自己想
5 分钟

不查笔记，先答 ——

  • 一个反应 A + B → C + D 的本质 = 哪些键变化了？反应物里的某些键消失了(过程 1)，产物里某些新键出现了(过程 2)。
  • 哪个过程吸热？哪个过程放热？
  • 总 ΔH 应该是哪两部分的差？如何用键能数据写出公式？

给一个核心猜测：反应 ΔH = "断反应物键吸的热" − "成产物键放的热" = Σ(反应物键能) − Σ(产物键能)。Card 2 推导这点 + 用 3 个例题练习。

STEP 2键能算法公式 + 推导
反应焓变的键能算法 ⭐⭐

\(\Delta H_反应 \approx \Sigma\)（反应物键能, 断键吸热）\(- \Sigma\)（产物键能, 成键放热）

数值正负: 反应物键能之和 > 产物键能之和 → ΔH > 0 (吸热); 反应物键能之和 < 产物键能之和 → ΔH < 0 (放热)

公式的盖斯定律推导:

用盖斯定律证明键能算法公式
考虑反应 A + B → C + D。盖斯定律允许我们选一条虚拟路径: A + B → (全部原子态) → C + D。
步骤 ①: A + B → 全部气态原子。这一步要断开所有反应物的化学键 → 吸热 = Σ Δ(反应物键能)(每根键吸的热相加)
步骤 ②: 全部气态原子 → C + D。这一步要形成所有产物的化学键 → 放热 = −Σ Δ(产物键能)(每根键放的热相加; 负号因为成键放热 ΔH < 0)
盖斯定律: ΔH反应 = ΔH①① + ΔH② = Σ(反应物键能) − Σ(产物键能) ✓
核心收获: 键能算法不是"新原理", 它是盖斯定律应用到"原子态"这条虚拟路径上的特殊情形。这就是为什么 prereq-recap 要先回顾盖斯定律。
图 1 · 键能算反应 ΔH 的"三层能级图"。最上方灰色虚线为全部气态原子态(参考最高能态, 所有化学键都已断开); 中部红色实线为反应物(中能态); 下方绿色实线为产物(最低能态, 因反应放热)。红色箭头表示步骤 ① 断键(反应物 → 原子态, 吸热 = +Σ反应物键能, 左侧引线标注); 绿色箭头表示步骤 ② 成键(原子态 → 产物, 放热 = −Σ产物键能, 右侧引线标注); 蓝色虚线箭头表示净 ΔH(反应物 → 产物的直接能量差, = ① + ②)。底部蓝色公式为键能算法核心公式 ΔH ≈ Σ(反应物键能) − Σ(产物键能)。
STEP 3三个典型例题 — 算反应 ΔH
思考点 3 · 用键能算 H₂(g) + Cl₂(g) → 2HCl(g) 的 ΔH
最简单的单键反应。给定键能数据(kJ/mol): H-H = 436, Cl-Cl = 243, H-Cl = 431。
用键能算法算 ΔH 反应, 判断放热还是吸热。
分析反应物 / 产物的键 ——
· 反应物 H₂ + Cl₂: 1 根 H-H + 1 根 Cl-Cl
· 产物 2HCl: 2 根 H-Cl

用公式 ΔH ≈ Σ(反应物键能) − Σ(产物键能):
· Σ(反应物键能) = 1 × 436 + 1 × 243 = 679 kJ/mol
· Σ(产物键能) = 2 × 431 = 862 kJ/mol
· ΔH ≈ 679 − 862 = −183 kJ/mol(放热, 与实测 −185 kJ/mol 一致 ✓)

核心收获: 反应放热, 因为产物的 H-Cl 键(431)比反应物的 H-H(436) 和 Cl-Cl(243)的平均更稳定(尤其 Cl-Cl 较弱 + 2 个稳定 H-Cl 形成)。键能算法的本质就是"比较反应物和产物键的总稳定度"。
在纸上: ① 列出反应物 / 产物的键; ② 算 Σ(反应物键能) 和 Σ(产物键能); ③ 计算 ΔH = 679 − 862 = −183 kJ/mol; ④ 一句话总结"反应放热是因为产物键比反应物键更稳定"。
思考点 4 · 用键能算 CH₄(g) + 2O₂(g) → CO₂(g) + 2H₂O(g) 的 ΔH
甲烷燃烧 — 但产物 H₂O 写成气态(键能算法的限制要求全气态)。给定键能数据(kJ/mol): C-H = 413, O=O = 498, C=O = 745, O-H = 463。
算 ΔH, 然后与燃烧热 ΔH燃(CH₄) = −890.3 kJ/mol(H₂O 液态)比较, 解释差额。
分析反应物 / 产物的键 ——
· 反应物 CH₄ + 2O₂: 4 根 C-H + 2 根 O=O
· 产物 CO₂ + 2H₂O(g): 2 根 C=O + 4 根 O-H

用公式:
· Σ(反应物键能) = 4 × 413 + 2 × 498 = 1652 + 996 = 2648 kJ/mol
· Σ(产物键能) = 2 × 745 + 4 × 463 = 1490 + 1852 = 3342 kJ/mol
· ΔH ≈ 2648 − 3342 = −694 kJ/mol(放热, 但比燃烧热 −890 小)

差额解释: 燃烧热 ΔH燃 = −890 是 H₂O 液态; 本题算的是 H₂O 气态。
· 差额 ≈ |−890| − |−694| = 196 kJ/mol(燃烧到液态比到气态多放 196)
· 2 mol H₂O 由气态液化放热 = 2 × 44 = 88 kJ/mol(每 mol 水液化放热 44, 即蒸发热的反过程)
· 把键能算结果"修正到液态" → −694 − 88 = −782 kJ/mol (仍比 −890 少 108)
· 剩下 108 kJ/mol ≈ 平均键能算法的固有误差(890 × 12% ≈ 107, 符合 ±5-10% 量级)
· 主要差额是键能算法的精度限制(用平均键能, 不严格); 液化修正 88 是次要贡献

核心收获: 键能算法精度 ±5-10%; 实测 / 盖斯 ΔHf° 精度 ±1%。键能法用于趋势判断 + 数据缺失时的工程估算, 不用于高精度。
在纸上: ① 列出反应物 / 产物的键; ② 算 Σ 和 ΔH = −694 kJ/mol; ③ 解释 −694 vs −890 差额 = 蒸发热 + 平均键能误差; ④ 一句话总结"键能算法精度 ±5-10%"。
注意"H₂O 液态 vs 气态"差额这点 — 典型问题(看是否理解键能算法的"全气态"限制 + 平均键能的精度局限)。
STEP 4键能算法的 4 个使用限制
限制原因违反后果
反应物 / 产物必须全气态键能定义在气态分子; 液固相内有额外分子间作用力(氢键 / 范德华力 / 离子键)液固相反应算出的 ΔH 偏离, 必须额外加蒸发 / 升华 / 溶解热修正
用平均键能(精度 ±5-10 kJ/mol)手册数据是多种含该键化合物的统计平均; 真实分子中每根键的离解能与平均键能差几十 kJ/mol计算的 ΔH 误差通常 ±5-10%(对几百-上千 kJ/mol 的反应)
不适用于离子反应 / 极性强反应键能针对共价键; 离子化合物 / 强极性反应涉及离子键 + 静电作用 + 水合等, 不能简单用键能加和用键能算 Na⁺ + Cl⁻ → NaCl 之类反应会严重偏离
必须完整断键 + 完整成键键能算法假设反应是"先全断 + 再全成"的虚拟路径(盖斯路径); 不考虑中间态或部分反应对慢反应 / 平衡态分析不直接适用; 反应机理细节(过渡态等)不在本算法内

典型适用场景: 全气态共价化合物的反应(如烃类燃烧 + 气相分解 + 气相合成)、初学者快速预判 ΔH 符号、手册中没有完整 ΔHf° 数据时的估算。

STEP 3综合例题 — 三种方法都算同一个反应
思考点 5 · 用三种方法都算 H₂(g) + ½O₂(g) → H₂O(g) 的 ΔH, 比较精度
这个反应数据齐全, 适合作为"三种方法都算一遍"的标杆。
给定数据 ——
· 键能(kJ/mol): H-H = 436, O=O = 498, O-H = 463
· ΔHf°(kJ/mol, 25°C): H₂(g) = 0, O₂(g) = 0, H₂O(g) = -241.8
· 实验: 氢气燃烧到水蒸气的实测反应热 ≈ -242 kJ/mol(经过 H₂O 液态 -286 + 蒸发 +44 修正得到)

用三种方法都算 ΔH, 然后比较精度。
三种方法 ——

方法 ① 键能算法:
· 反应物键: 1 H-H + ½ O=O → Σ = 436 + ½ × 498 = 436 + 249 = 685 kJ/mol
· 产物键: 2 O-H → Σ = 2 × 463 = 926 kJ/mol
· ΔH ≈ 685 − 926 = −241 kJ/mol

方法 ② 盖斯定律 + ΔHf°:
· ΔH = Σν·ΔHf°(产物) − Σν·ΔHf°(反应物)
· = 1 × (-241.8) − [1 × 0 + ½ × 0]
· = −241.8 kJ/mol

方法 ③ 实验直接测量:
· 弹式量热计测 H₂ 燃烧 (生成 H₂O 液态) → ΔH = -285.8 kJ/mol
· 加 H₂O 蒸发热修正 (液态 → 气态 +44 kJ/mol per mol) → -285.8 + 44 = −241.8 kJ/mol(转气态)

三方法结果比较:
· 键能: -241(估算)
· ΔHf°: -241.8(高精)
· 实验: -241.8(高精)
· 三者一致 ✓ — 键能法在本反应里"运气好", 误差 < 1%; 实际场景中键能算法误差通常 5-10%。

核心收获: 同一个反应可以用三种独立方法算出基本一致的 ΔH — 这是反应焓变的"路径无关性"的有力证据(盖斯定律). 三种方法是互补的: 实验测 + ΔHf° 高精, 键能法适合数据缺失时的快速估算或趋势预判。
在纸上: ① 用三种方法分别算 ΔH; ② 列表对比三个结果, 看差异; ③ 一句话总结"三种方法是同一反应能量的不同入口, 互补不替代"。
"用三种方法都算一遍"是物理化学解题的有力工具 — 当某一种结果异常时, 用另两种方法验证, 找出哪里错了。
